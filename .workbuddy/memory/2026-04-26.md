## 支付宝支付集成（2026-04-26）
- .env 已配置：ALIPAY_APP_ID、ALIPAY_PRIVATE_KEY、ALIPAY_PUBLIC_KEY、ALIPAY_NOTIFY_URL、ALIPAY_RETURN_URL
- 新增 payment_service.py：
  - PACKAGES 套餐：pkg_10(10元/100积分)、pkg_50(50元/600积分)、pkg_100(100元/1500积分)
  - create_order(user_id, package_id)：创建支付宝 page.pay 订单，写入 Redis order:{order_id}（TTL 2h）
  - verify_notify(params)：RSA2 验签（SDK 懒加载，无 SDK 时返回 False）
  - handle_paid_notify(params)：幂等处理，更新订单状态 + 调用 billing_service.grant_credits 发放积分
  - get_order(order_id)：查询 Redis 订单详情
  - set_redis_client(rc)：由 main.py 注入
  - SDK 懒加载策略：无 alipay-sdk-python 时抛 RuntimeError → main.py 转为 HTTP 503
- 修改 main.py：
  - 导入 payment_service，注入 payment_service.set_redis_client(redis_client)
  - 新增 POST /api/order/create：JWT 认证，创建订单返回支付 URL
  - 新增 POST /api/order/webhook：支付宝异步通知，验签后发放积分，返回纯文本 "success"
  - 新增 GET /api/order/status/{order_id}：JWT 认证，仅查自己订单
  - 新增 Pydantic 模型：CreateOrderRequest/Response、OrderStatusResponse
- 修改 requirements.txt：加入 alipay-sdk-python>=3.3.398（生产 Docker 安装）
